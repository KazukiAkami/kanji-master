# 問題集の追加手順

漢字マスターに新しい問題集（定例試験など）を追加する手順です。
**編集するのは `kanji-data.js` だけ**です。`index.html` や `app.js` など他のファイルは触りません。
**クオートやバッククオートを手で書く必要はありません。**

---

## 1. `kanji-data.js` を開く

テキストエディタで `kanji-data.js` を開きます。
ファイルの中に、すべての問題集を入れた `KANJI_DATA` という文字列があります。

```
const KANJI_DATA = `
## 定例試験_202605
習い事をヤめる。,辞める
お店をイテンする。,移転

## 定例試験_202606
【読み】店を構える,店をカマえる
`;
```

先頭と末尾を囲んでいる **バッククオート `` ` `` の1組は触らない**でください。
新しい問題集は、この囲みの**内側**に追記します。

---

## 2. 問題集を1つ追記する

囲みの内側の好きな場所（末尾が分かりやすい）に、次の形で書き足します。

```
## 定例試験（2026/07/01）
駅前のビルへお店をイテンする。,移転
夕方になって、お菓子がハンガクになった。,半額
```

- **`## タイトル`** の行で新しい問題集が始まります。
  このタイトルがホーム画面のプルダウンに表示され、**進捗の保存キー**にもなります。
- それ以降の行は **`問題,答え`**。**最初のカンマ**で問題と答えに分けます。
- 空行は無視されるので、問題集の間を1行空けると見やすくなります。
- 見出し行（`question,answer` など）は**不要**です。書くとそれが1問目になってしまいます。

追記して保存すると、ホーム画面のプルダウンに自動で増えます。

---

## 3. 問題の書き方（表記ルール）

このアプリの問題集は、次の書き方で統一しています。

- **書き取り問題** … 漢字に直してほしい部分を問題文の中で **カタカナ** で書き、答え列にその漢字（送り仮名はひらがな）を書きます。
  例: `習い事をヤめる。,辞める` ／ `お店をイテンする。,移転`
  送り仮名がある場合は、送り仮名はひらがなのまま、直す音だけカタカナにします。
  例: `山の風がココロヨい気分だった。,快い`
- **読み問題** … 問題文の頭に `【読み】` を付け、漢字を問題文に出し、その読みを答え列で **カタカナ** で書きます。
  例: `【読み】店を構える,店をカマえる`
- **四字熟語** … 問題文の頭に `【四字熟語】` を付け、意味を問題文に、四字熟語を答え列に書きます。
  例: `【四字熟語】言葉を口にしなくても相手に気持ちが伝わる事,以心伝心`

---

## 4. 注意点

- **問題文の中の読点は全角「、」を使う。** 半角カンマ `,` は問題と答えの区切りに使われるため、問題文に混ぜると答えが欠けます。
- **タイトル（`## …`）は問題集ごとに重複させない。** 同じタイトルが2つあると進捗が混ざります。
- **公開後にタイトルを変えない。** 進捗（`localStorage` の `kanjiProgress:<タイトル>`。まちがえた問題・正解率・「つづきから」の途中状態を保持）が別物として扱われ、リセットされたように見えます。
- **囲みのバッククオート `` ` `` は触らない。データに `` ` `` と `${` を含めない。** 文字列が壊れます（漢字書き取り問題ではまず出てきません）。
- 問題は **書いた順** に出題されます（通常モードはシャッフルなし）。

---

## 5. 動作確認

`index.html` をブラウザで開き、

1. ホーム画面のプルダウンに新しいタイトルが表示されること（問題数も `（◯問）` と出ます）
2. 選ぶと問題が出題されること
3. 1問目が見出し行（`question` / `answer`）になっていないこと

を確認します。表示されない・問題がおかしい場合は、次をチェックしてください。

- `## タイトル` の行が「`## 」`（シャープ2つ＋半角スペース）で始まっているか
- 問題行に半角カンマ `,` が1つ入っているか（区切りが無い行は無視されます）
- 問題文の読点を全角「、」にしているか
- ブラウザの再読み込み（キャッシュが残っている場合は強制再読み込み）
