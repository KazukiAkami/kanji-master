# 問題集の追加手順

漢字マスターに新しい問題集（定例試験など）を追加する手順です。
**編集するのは `kanji-data.js` だけ**です。`index.html` や他のファイルは触りません。

---

## 1. 元になる問題データを用意する

問題と答えを2列で用意します。形式は次のどちらでもOKです。

- **表計算ソフト（Excel・スプレッドシート等）** … セルを2列で選択してコピー（タブ区切りになります）
- **CSVテキスト** … `問題,答え` のようにカンマ区切りで書く

1行目は見出し行 `question,answer`（または `問題,答え`）にしてください。
この見出し行だけ自動でスキップされます。

例（CSV）：

```
question,answer
部活動が忙しくなったので、習い事をヤめる。,辞める
駅前のビルへお店をイテンする。,移転
```

> 見出しを付けない場合は1行目からそのまま問題として扱われます。
> 独自の見出し語（「漢字テスト」など）を書くと、それが問題に混ざるので注意してください。

### 表記のルール（重要）

このアプリの問題集は、次の書き方で統一しています。

- **書き取り問題** … 漢字に直してほしい部分を問題文の中で **カタカナ** で書き、答え列にその漢字（送り仮名はひらがな）を書きます。
  例: `習い事をヤめる。,辞める` ／ `お店をイテンする。,移転`
  送り仮名がある場合は、送り仮名はひらがなのまま、直す音だけカタカナにします。
  例: `山の風がココロヨい気分だった。,快い`
- **読み問題** … 問題文の頭に `【読み】` を付け、漢字を問題文に出し、その読みを答え列で **カタカナ** で書きます（読ませる対象だけカタカナ、それ以外は元のまま）。
  例: `【読み】店を構える,店をカマえる`
- **四字熟語** … 問題文の頭に `【四字熟語】` を付け、意味を問題文に、四字熟語を答え列に書きます。
  例: `【四字熟語】言葉を口にしなくても相手に気持ちが伝わる事,以心伝心`

## 2. `kanji-data.js` を開く

テキストエディタで `kanji-data.js` を開きます。
ファイル内の `kanjiData` 配列の中に、次の目印があります。

```js
// ★ ここから下に新しい問題集を追記する ★★★★★★★★★★★★★★★★★★★★★★
// {
//     id: '2026-07-teirei',
//     title: '定例試験（2026/07/01）',
//     csv: `question,answer
// ここに問題,ここに答え`
// },
// ★ ここまで ★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★
```

---

## 3. 1ブロック追記する

`★ ここから ★` と `★ ここまで ★` の間に、次の形のブロックを1つ追記します。

```js
{
    id: '2026-07-teirei',
    title: '定例試験（2026/07/01）',
    csv: `question,answer
駅前のビルへお店をイテンする。,移転
夕方になって、お菓子がハンガクになった。,半額`
},
```

### 各項目の意味

| 項目 | 説明 | 例 |
|------|------|-----|
| `id` | 内部で使う識別子。英数字とハイフン。**進捗の保存キーになるため、一度公開したら変えない**。問題集ごとに重複しない値にする | `'2026-07-teirei'` |
| `title` | ホーム画面のプルダウンに表示される名前 | `'定例試験（2026/07/01）'` |
| `csv` | 問題データ。バッククオート `` ` `` と `` ` `` の間に貼る。1行目は見出し行 | （上の例を参照） |

---

## 4. 注意点

- **`id` は問題集ごとに重複させない。** 既存と同じ `id` を使うと進捗が混ざります。
- **公開後に `id` を変えない。** 進捗（`localStorage` の `kanjiProgress:<id>`）が別物として扱われ、リセットされたように見えます。
- **バッククオート `` ` `` と `${` をデータに含めない。** テンプレートリテラルが壊れます。万一データに出てくる場合は事前に取り除いてください。
- 各ブロックの末尾は **カンマ `,`** で区切ります（最後のブロックは付けても付けなくても可）。
- 問題は **配列に書いた順** に出題されます（通常モードはシャッフルなし）。

---

## 5. 動作確認

`index.html` をブラウザで開き、

1. ホーム画面のプルダウンに新しい `title` が表示されること
2. 選ぶと問題が出題されること
3. 1問目が見出し行（`question` / `answer`）になっていないこと

を確認します。表示されない・問題がおかしい場合は、次をチェックしてください。

- ブロックの前後のカンマ・カッコの対応が取れているか
- バッククオートが正しく閉じているか
- ブラウザの再読み込み（キャッシュが残っている場合は強制再読み込み）
